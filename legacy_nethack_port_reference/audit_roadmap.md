## 5. 이식 가이드라인

### 5.1 이식 시 체크리스트

- [ ] `cargo build` 에러 0개
- [ ] `cargo test` 전체 통과
- [ ] 새로운 `pub fn`이 최소 1곳에서 호출 확인
- [ ] 매직넘버 상수 분리 확인
- [ ] 주석 전체 한국어 확인
- [ ] `audit_roadmap.md`, `CHANGELOG.md`, `IMPLEMENTATION_SUMMARY.md` 동기화

### 5.2 이식 원칙

1. **호출부 우선 원칙**: 유틸 함수 이식 시 반드시 호출하는 시스템도 함께 구현.
2. **ECS 래퍼 의무화**: 순수 함수만 이식하고 ECS 래퍼를 구현하지 않으면 미완성.
3. **매직넘버 상수화**: 원본 NetHack의 하드코딩 확률값은 `const`로 분리.
4. **Pure Result 패턴**: 모든 신규 함수는 순수 결과 반환 체계로 작성.
5. **패닉 제로**: `unwrap()`/`expect()` 사용 금지.
